ResidueTemplate.getRotamericDihedrals(phi, psi, rotNum, dihedralNum)¶

Returns the dihedral angle dihedralNum, for rotamer rotNum, for the backbone closest to the values phi and psi. If the rotamer library is backbone independent, then phi and psi are ignored.

Parameters:
  • phi (float) – The backbone phi angle
  • psi (float) – The backbone psi angle
  • rotNum (int) – The rotamer’s index number
  • dihedralNum (int) – The dihedral’s index number.

ResidueTemplate.setNumberOfPhiPsiBins(numberOfPhiPsiBins)¶

Number of “bins” for Phi/Psi angles for backbone dependent rotamer libraries. For example, in the Dunbrack library there are 37 bins.

Parameters:numberOfPhiPsiBins (int) –
ResidueTemplate.setRLphiPsiResolution(phiPsiResolution)¶

Resolution of backbone “bins” in the rotamer library. For example in a backbone independent rotamer library this should be “360”. For the Dunbrack rotamer library this is 10. Set in the corresponding parser of the rotamer library.

Parameters:phiPsiResolution (float) –
ResidueTemplate.initializeRotamerArrays()¶

Initialize the numRotamers and rotamericDihedrals arrays to the number of bins according to each rotamer library.

ResidueTemplate.setNumRotamers(numRotamers, phiBin, psiBin)¶

PGC 2015: Sets the number of rotamers for a residue template; phi and psi are only used when using a backbone dependent rotamer library.

Parameters:
  • numRotamers (int) – Number of rotamers
  • phiBin (int) – Bin in the rotamer array where the rotameric dihedrals will be stored.
  • psiBin (int) – Bin in the rotamer array where the rotameric dihedrals will be stored.
